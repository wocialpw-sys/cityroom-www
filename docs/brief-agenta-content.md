## Sekcje do wypełnienia

### 1. Misja agenta
Co produkuje, dla kogo, w jakim systemie (cluster pages → blog → email follow‑up).

### 2. Audiencja
Persona prymarna: "Zmęczony Landlord" (5 push forces — patrz `cityroom_strategy_foundation.md`). Język drugiej osoby liczby pojedynczej, "Ty".

### 3. Voice & tone
- Profesjonalny‑konwersacyjny, spokojny, zaufany
- Bez wykrzykników, bez emoji, bez startup vocabulary
- Czas teraźniejszy dokonany ("dostarczamy", nie "gwarantujemy")
- Sentence case wszędzie
- Polskie typo (spacje w liczbach, low‑high cudzysłowy, em‑dashy)

### 4. Brand rules
- Bez liczby "7" (lat / osób — nieaktualne)
- Bez nazw konkurencji w copy ("agencje prowizyjne", nie Fiesta/InHarbor)
- Dwa wymiary UM "Regularna niezawodność" zawsze razem (codzienna konsekwencja + odporność na stres)

### 5. Struktura pisania (GEO‑oriented)
- Pierwsze 200 słów = direct answer (TLDR)
- Chunki 134–167 słów per H2/H3
- Listicle format dla blog
- Triple JSON‑LD schema (Article + ItemList + FAQPage)
- Inline citations + linki do źródeł
- Specyficzne, datowane, weryfikowalne claims

### 6. Proof points (single source of truth)
Patrz `proof-points.md` — agent powinien losować z tego katalogu, nie wymyślać.

### 7. FAQ patterns
3–5 pytań na dole każdej cluster page (FAQPage schema). Patrz `faq-master.md` — dystrybucja po stronach.

### 8. Per‑typ‑strony specyfikacja
- Pillar
- Cluster page (P1/P2/P3)
- Listicle blog
- Comparative blog
- Definitional content
- FAQ page

### 9. Iteracja briefu
Pierwszy agent na jednej cluster page = test. Iteracja briefu po review. Skalowanie dopiero po stabilizacji.

### 10. Walidacja outputu agenta
Checklist przed merge — patrz `cityroom-geo-strategy` skill, sekcja 15 (Quick reference checklist).
